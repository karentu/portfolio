A/B Testing

“A picture says a thousand words”, but is that too many?

“A picture says a thousand words”, but is that too many? I used AB and User testing to examine the possibility that a bad picture is worse than no picture at all. I created two versions of the same generic taxi website. Version A contains the exact same content as Version B, except without the vague (bad) taxi pictures. Fundamentally, the testing asks a question about user priorities: do people care more about the visual presentation of a page or the quality of the content?

View the website!

I surveyed 54 of my classmates and obstained the following statistics when tracking the following for statistics: dwell time, click-through rate, return rate, and time to click.

The statistical tests were inconclusive and found no significant difference between website version and B.

User Testing

Following the AB Testing, I wanted to figure out what kind of information was most important to users.

Hypothesis: Users’ are primarily concerned about price when booking a cab; all other factors that influence the choice of a cab are secondary to that primary concern.

I designed the mobile prototype with this hypothesis in mind and ordered the filtering of the different taxi companies accordingly.

See the prototype!

Tools

I used HTML and CSS to create the A and B versions of the web page hosted on HerokuApp for the primary expirement. Then, I used Figma to mock up a mobile app as a follow up confirmation experiment through user testing on UserTesting.com.

Takeaways

Although both User Testing and AB Testing required me to evaluate a hypothesis of some kind, User Testing allowed for much more open-ended responses, revealing considerations that I hadn’t considered before. On the other hand, A/B Testing provides a much more rigorous standard of proof that can distinguish effectively the subconscious effects of two relatively similar designs.

In future designs, it may be helpful to employ User Testing in order to do more exploratory hypothesis testing and use A/B Testing to gain design specificity after the general direction of the design has been determined by User Testing.
